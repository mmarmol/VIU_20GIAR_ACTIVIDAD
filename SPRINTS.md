## 🗺️ Roadmap por Sprints – Dashboard Financiero

### Sprint 0 – Preparación (opcional)

**Duración:** 0,5 día  
_Este sprint no es funcional, pero es recomendable._

**Objetivo**  
Dejar listo el entorno de trabajo y el repositorio.

**Actividades**
- Crear repositorio en GitHub
- Definir ramas (`main`, `dev`)
- Configurar `.gitignore`
- Definir convenciones de commits

✅ **Resultado**  
Repositorio listo para comenzar el Sprint 1.

---

### Sprint 1 – Base del proyecto y calidad

**Duración:** 1 semana  
**Objetivo:** Crear una base sólida, escalable y profesional.

**Tickets incluidos**
- ✅ TICK-01: Inicializar proyecto
- ✅ TICK-02: Configurar ESLint
- ✅ TICK-03: Configurar Vitest

**Entregables**
- Aplicación React creada con Vite
- Estructura de carpetas definida
- Lint y tests ejecutándose por CLI
- Test de ejemplo funcionando

🎯 **Valor aportado**  
Base técnica profesional desde el inicio del proyecto.

---

### Sprint 2 – Autenticación y navegación

**Duración:** 1 semana  
**Objetivo:** Controlar el acceso y el flujo de usuario.

**Tickets incluidos**
- ✅ TICK-04: Implementar login
- ✅ TICK-05: Implementar logout
- ✅ TICK-06: Proteger rutas

**Entregables**
- Pantalla de login funcional
- Contexto de autenticación
- Rutas protegidas
- Redirecciones correctas según el estado de autenticación

🎯 **Valor aportado**  
Dashboard seguro y flujo de usuario realista.

---

### Sprint 3 – Layout y datos

**Duración:** 1 semana  
**Objetivo:** Construir la base visual del dashboard y la obtención de datos.

**Tickets incluidos**
- ✅ TICK-07: Crear layout principal
- ✅ TICK-08: Integrar API financiera

**Entregables**
- Layout responsive del dashboard
- Navbar con botón de logout
- Servicio de datos desacoplado
- Datos reales con fallback mock
- Gestión de estados de carga y error

🎯 **Valor aportado**  
El proyecto ya se visualiza como un dashboard real y consume datos financieros.

---

### Sprint 4 – Visualización y filtros

**Duración:** 1 semana  
**Objetivo:** Implementar el núcleo funcional del dashboard financiero.

**Tickets incluidos**
- ✅ TICK-09: Gráfico de precio
- ✅ TICK-10: Gráfico de volumen
- ✅ TICK-11: Filtro por ticker
- ✅ TICK-12: Filtro por rango temporal

**Entregables**
- Gráficos sincronizados
- Filtros interactivos
- Actualización dinámica de los datos
- Experiencia de usuario clara y usable

🎯 **Valor aportado**  
Dashboard completo y listo para ser utilizado o demostrado.

---

### Sprint 5 – Tests, CI/CD y documentación

**Duración:** 1 semana  
**Objetivo:** Dejar el proyecto preparado para producción y presentación.

**Tickets incluidos**
- ✅ TICK-13: Tests de autenticación
- ✅ TICK-14: Tests de dashboard
- ✅ TICK-15: Pipeline CI (dev)
- ✅ TICK-16: Pipeline CD (main)
- ✅ TICK-17: Documentar arquitectura
- ✅ TICK-18: Documentar README

**Entregables**
- Tests automáticos completos
- Pipeline de CI en la rama `dev`
- Deploy automático en GitHub Pages desde `main`
- Aplicación accesible públicamente
- Documentación técnica y de uso

🎯 **Valor aportado**  
Proyecto plenamente profesional, documentado y desplegado.

---

## 📊 Resumen del roadmap

- **Sprint 1:** Setup y calidad → Base técnica sólida
- **Sprint 2:** Autenticación → Aplicación segura
- **Sprint 3:** Layout y datos → Dashboard funcional
- **Sprint 4:** Gráficos y filtros → Valor funcional completo
- **Sprint 5:** CI/CD y documentación → Proyecto desplegado
